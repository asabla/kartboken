# Kartboken v1.0

Status: release candidate complete.

Version 1.0 is a useful public map, not the end of the product. It proves that a small, static, source-backed catalog can support enjoyable discovery without accounts or runtime services.

## Acceptance boundary

- Browse every published place on an interactive map and in an accessible list.
- Search names, descriptions, addresses, localities, regions, categories, and tags.
- Filter by place category and how far a place is worth travelling.
- Select a place from either map or list and preserve useful state in the URL.
- Show recommendation, official, and geodata sources for every place.
- Build to static files and allow the basemap style to be replaced by configuration.
- Run catalog validation, unit tests, fuzzy properties, and browser tests in CI.
- Document local development, static hosting, map hosting, and catalog curation.

## Explicitly later

- Accounts, saved places, votes, public submissions, and administration.
- Route planning or directions between places.
- Automatically scraped or automatically published records.
- Live opening hours, ratings, price levels, and availability.
- A runtime database, application API, or background worker.
